# v1 → v2 production DB migration

Use this when migrating the live **v1 production database** (Railway, ~15.5k users)
to the v2 schema.

## TL;DR

The v1 DB's `_prisma_migrations` history is a **clean prefix** of this repo's —
applied through `20260613133500_add_onboarding_completion`. So there is **no
divergence and no baseline/`migrate resolve` needed**. Just run:

```bash
DATABASE_URL="postgresql://…/railway" ./scripts/migration/run-v1-to-v2.sh
```

That backs up the DB, runs `prisma migrate deploy` (applies the 7 pending v2
migrations in order), and tells you how to re-seed the shop.

## What gets applied (7 pending migrations)

| migration | effect |
|---|---|
| `20260614223726_v2_progression_economy` | adds Syrup/`ticketBalance`(=3), `lives`(=5), `xp`, streaks; `LevelProgress`, power-ups, cosmetics, boosts, badges, `Winning`, `RoundEntry`, `TicketLedger` |
| `20260615061321_v2_partner_offers_season_pass` | `PartnerOffer(+Claim)`, `SeasonPassClaim`, `DailyRewardClaim` |
| `20260617052902_add_question_template_category` | `QuestionTemplate.category` |
| `20260617120000_add_analytics_events` | `AnalyticsEvent` |
| `20260618120000_v2_league_cohorts` | `League`, `LeagueCohort`, `LeagueMember` |
| `20260618130000_league_reward_ticket_reason` | enum value |
| `20260618140000_remove_chat` | **DROP TABLE "Chat"** (≈181 rows) |

## Safety

- ✅ No `ADD COLUMN NOT NULL` without a default → won't fail on existing rows.
- ✅ No `DROP COLUMN`; no unique index on a populated table.
- ✅ `GameEntry` needs no changes — v1 already has the merkle/rank/prize columns.
- ⚠️ Only data loss is the intentional `DROP TABLE "Chat"`. The pg_dump backup
  the script takes first preserves those rows.
- ℹ️ Existing users get `ticketBalance=3` — that's **Syrup** (off-chain soft
  currency), not USDC tournament tickets. Harmless welcome grant. To grant 0
  instead, edit the `DEFAULT 3` in `20260614223726_v2_progression_economy`
  before deploying.

## Files

- `run-v1-to-v2.sh` — the runnable migration (backup → deploy → seed hint).
- `v1_to_v2.reference.sql` — the **net** schema delta, generated by
  `prisma migrate diff`. For **review/dry-read only** — do NOT apply it directly;
  `migrate deploy` is the real path (it keeps `_prisma_migrations` correct).

## After migrating

```bash
pnpm prisma migrate status                 # should show "up to date"
# re-seed the shop (ShopItem is created empty):
DATABASE_URL="$DATABASE_URL" node --env-file=.env.production --import tsx scripts/seed-v2-shop.ts
```

## Rollback

```bash
pg_restore --clean --no-owner -d "$DATABASE_URL" db_backup_v1_pre_v2_<stamp>.dump
```
